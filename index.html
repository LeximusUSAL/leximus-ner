📖 ¿Qué es LexiMus NER?

LexiMus NER es un recurso de reconocimiento de entidades nombradas (NER) especializado en músicos, compositores, intérpretes, cantantes y agrupaciones musicales de la prensa y revistas españolas históricas.

Está formado por dos elementos: un CSV con 984 entidades únicas (1.035 filas incluyendo alias y variantes ortográficas) curadas y revisadas manualmente a partir del corpus de prensa histórica de principios del siglo XX de LexiMus USAL, y un script Python que busca esas entidades en cualquier corpus de archivos .txt usando expresiones regulares. No requiere spaCy ni ninguna biblioteca de terceros.

✓ El script funciona con Python 3.7 o superior sin instalar nada más. Solo necesitas descargar los dos archivos y apuntarlo a tu corpus.

🏷️ Categorías de entidades

COMPOSITOR

Compositores activos en prensa española: Falla, Albéniz, Granados, Brahms, Mozart…

INTERPRETE

Instrumentistas y directores de orquesta: pianistas, violinistas, guitarristas, concertistas, directores…

CANTANTE

Voces líricas y populares: sopranos, tenores, cantaoras, cupletistas…

AGRUPACION

Orquestas, bandas, coros, cuartetos, coblas y otras formaciones colectivas.

📊 Estructura del CSV

El archivo entidades_ner_leximus.csv tiene seis columnas:

Columna Descripción Ejemplo
texto Forma encontrada en el corpus (puede tener variantes ortográficas) Falla
etiqueta Tipo de entidad COMPOSITOR
variante_limpia Forma normalizada sin apodos ni erratas Manuel de Falla
instrumento_rol Instrumento principal o rol artístico (opcional) piano
notas Notas de curación (apodos, restricciones de contexto…) Apodo: «Don Manuel»
mismo_que Nombre canónico cuando texto es un alias Manuel de Falla
ℹ️ Puedes añadir nuevas entidades al CSV con cualquier editor de hojas de cálculo. El script lee el CSV en cada ejecución, así que los cambios se aplican de inmediato.
ℹ️ ¿Por qué el CSV tiene 1.035 filas si hay 984 entidades únicas? Porque una misma persona o agrupación puede aparecer en el corpus con grafías distintas (errores de OCR, variantes ortográficas, apellido solo, apodos…). Cada variante ocupa su propia fila para que el script pueda detectarla, y la columna mismo_que la vincula a la forma canónica. Las 51 filas extra son esos alias: por ejemplo, banda Creatone → Banda Creatore, o Orfeón Gracense → Orfeón Gracienc.

🚀 Cómo usar el script

Descarga los dos archivos

Necesitas buscar_entidades_leximus.py y entidades_ner_leximus.csv en la misma carpeta. 

# Opción A — clonar el repositorio completo
git clone https://github.com/LeximusUSAL/leximus-ner.git
cd leximus-ner

# Opción B — descargar solo los dos archivos esenciales
curl -O https://raw.githubusercontent.com/LeximusUSAL/leximus-ner/main/buscar_entidades_leximus.py
curl -O https://raw.githubusercontent.com/LeximusUSAL/leximus-ner/main/entidades_ner_leximus.csv

Prepara tu corpus

El script busca en archivos .txt con codificación UTF-8. Pueden estar todos en una carpeta o en subcarpetas (se busca de forma recursiva).

Si tus archivos tienen nombre con fecha (ondas_1927-03-15.txt), el script extrae la fecha automáticamente para cada mención.

⚠️ Los archivos deben estar en UTF-8. Si tienes textos en Latin-1 o Windows-1252, conviértelos primero con iconv -f latin1 -t utf-8 archivo.txt > archivo_utf8.txt

Ejecuta el script

Abre una terminal, navega a la carpeta donde descargaste los archivos y ejecuta:

python3 buscar_entidades_leximus.py

El script te pedirá la ruta a tu carpeta de corpus:

============================================================

Proyecto PID2022-139589NB-C33 (USAL/UCM/UR)
============================================================

  CSV cargado: 1035 variantes → 984 entidades únicas

Introduce la ruta a la carpeta con los archivos .txt del corpus.
Ejemplo: /Users/maria/corpus/ondas_txt

Ruta al corpus:

Consulta los resultados

El script genera tres archivos en la carpeta del corpus:

resultados_leximus.txt

Listado de texto

Resumen legible con todas las entidades, variantes detectadas y número de menciones por archivo.

resultados_leximus.json

Datos completos

JSON con todas las menciones, contextos, fechas y variantes. Ideal para análisis posteriores.

resultados_leximus.html

Revisión interactiva

Abre en el navegador. Permite marcar entidades como correctas o falsos positivos y exportar la revisión.

✏️ Cómo ampliar el CSV con nuevas entidades

Abre entidades_ner_leximus.csv con Excel, LibreOffice Calc o cualquier editor de texto y añade una fila por cada nueva entidad:

textoetiquetavariante_limpiainstrumento_rolnotasmismo_que
Pilar Lorengar CANTANTE Pilar Lorengar soprano
Lorengar CANTANTE Pilar Lorengar Pilar Lorengar
✓ Cuando el apellido solo puede referirse a dos personas distintas, añade ambas con el mismo texto pero diferente mismo_que — el script detectará la más larga primero y no solapará coincidencias.
